## 1. 软件概述

### 1.1 软件名称及版本
**软件名称**：Smart QC Tools 工业级质量控制软件  
**版本号**：V1.0  
**功能模块**：分布直方图分析模块 (Statistical Histogram Module)

### 1.2 开发背景与用途
分布直方图（Histogram）是质量管理中用于分析连续过程数据分布形态的核心工具。本模块旨在解决普通绘图工具无法处理规格限（USL/LSL）对比、无法自动拟合正态曲线以及分组逻辑不透明等问题。通过自研的 **Sturges/Doane 混合分组算法**，本软件能够自动为不同规模的数据集计算最优箱数（Bins），并实时叠加过程能力评价所需的关键统计参数，广泛应用于制程能力（Cp/Cpk）预判和异常分布识别。

## 3. 核心功能及操作规程

### 3.1 统计参数自动化配置
**功能描述**：系统具备强大的后端统计引擎，可在数据录入的同时完成复杂的聚合计算。

**操作规程**：
1. **分组策略控制**：
   - 默认采用 `auto` (自动) 模式，系统将根据样本量 $N$ 的对数关系自动确定组数。
   - 用户亦可手动输入正整数，强制设定为固定组数（例如 10 组、15 组），以观察不同粒度下的数据形态。
2. **正态拟合开关**：勾选 `ShowCurve` 后，系统将计算样本均值 $\mu$ 和标准差 $\sigma$，并绘制高斯分布曲线，辅助用户判断数据是否服从正态分布。

### 3.2 规格限 (Specification Limits) 交互
**功能描述**：通过在画布上绘制垂直基准线，直观展示产品规格要求。

**操作规程**：
- **USL (上规格限)**：输入图纸标称的最大允许值。
- **LSL (下规格限)**：输入图纸标称的最小允许值。
- **Target (目标值)**：输入设计中心值。
- **视觉反馈**：系统将分别以红/绿虚线显示这些界限。若数据柱状体越过 USL/LSL，即暗示制程存在超差风险。

### 3.3 DSL 语法编辑器操作 (Expert Console)
**语法规范**：
- `USL: [Number]`：设置上规格限。
- `LSL: [Number]`：设置下规格限。
- `Bins: [auto/Number]`：控制频数分布的组数。
- `ShowCurve: [true/false]`：控制拟合曲线显示。
- `- [Value]`：定义原始测量点。

## 4. 统计逻辑与算法优势

1. **动态分组算法**：内置最优组宽计算模型，避免了因组数过少导致的“信息掩盖”或组数过多导致的“过于分散”。
2. **高保真拟合引擎**：基于 JavaScript 数值计算库实现的概率密度函数（PDF）绘制，确保曲线平滑且符合统计理论。
3. **多维色彩方案**：支持 `Color[Bar]` 和 `Color[Curve]` 独立配置，提升在不同工业光照条件下的界面辨识度。
